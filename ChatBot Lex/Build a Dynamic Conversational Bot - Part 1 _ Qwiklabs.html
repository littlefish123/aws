Lab Overview

In this lab, you will use Amazon Lex to create and build a conversational bot that understands your requests and asks for more input if needed. If all required slots have been filled, the bot will ask for confirmation before triggering an action.

  • Create a bot by using Amazon Lex through the AWS Management Console.
  • Configure your bot to receive product order requests and assist you if needed.

By the end of this lab, you will be able to:

  • Create and build a conversational bot with Amazon Lex by using the AWS Management Console
  • Understand how to build an interaction model using utterances, intents, and slots

Technical Knowledge Prerequisites

This lab requires:

  • Access to a notebook computer with Wi-Fi and Microsoft Windows, Mac OS X, or Linux (Ubuntu, SuSE, or Red Hat)
  • An Internet browser such as Chrome, Firefox, or IE9 (previous versions of Internet Explorer are not supported). We recommend to use Chrome or Firefox.

Task 1: Create an Amazon Lex bot

In this task, you will create the initial version of your bot.

  1. In the AWS Management Console, ensure that the region is set to Oregon (us-west-2) in the right corner of the navigation bar.

  2. On the Service menu, click Amazon Lex.

You can also locate the services by typing its name in the search bar.

  1. If a Get Started page appears, click it.

  2. On the Create your Lex bot page, click Custom bot then configure:

  • Bot name:
  • Output voice: Choose a voice
  • Session timeout: min
  • COPPA: No

You will create a custom bot that lets you interact with it using voice commands. COPPA is not required because you are not going to deploy the bot on a page that is under COPPA regulation.

  1. Click Create

This will create a bot and will redirect you to the bot configuration page.

Task 4: Add Prompts to Your Slots

In this task, you will configure your bot to request the value for a required slot, and then complement your slots with Prompts and Corresponding utterances.

Add Prompts for ProductType

  1. In the Slots section, click on the settings icon for your productType.

This opens a new configuration window.

  1. Add the following prompts by:
  • Entering each one into the Prompts box one at a time
  • Clicking 
What would you like to order?

What kind of product would you like to order?

  1. Remove the default prompt e.g. What city? by clicking on .

  2. For Corresponding utterances, add:

I would like to order {productType}

  1. Click Save

Add Prompts for ProductFlavor

  1. In the Slots section, click on the settings icon for your productFlavor.

  2. For Prompts add:

Task 5: Add a Confirmation Prompt

In this task, you will add a Confirmation prompt to test your bot.

  1. Expand Confirmation prompt.

  2. Select Confirmation prompt, then configure:

  • Confirm:
Are you sure you want to order {orderQuantity} cups of {productFlavor} {productType}?
  • Cancel (if the user says "no"):
Okay. Your order will not be placed.

  1. Scroll to the bottom of the screen, then click Save Intent

Task 6: Build and Test Your Bot

In this task, you will build your bot and do some tests.

  1. At the top of the screen, click Build

  2. In the Build your bot pop-up, click Build

If you receive any errors during the build process, double-check to make sure there are spaces on both sides of the slots in your Sample utterances.

  1. Close the myBot build was successful message if it appears.

  2. On the right side of the lex console look for Test Bot window, if Test Bot window doesn't appear, click Test Chatbot to expand Test Bot window.

  3. Check error handling by sending a few random strings in the chat window. Your bot should answer with randomly picked phrases from the two Clarification prompts.
    After the third attempt, you should receive the Hang-up phrase (), which ends the session.

  1. Ask your bot for support by sending Since the help intent has no slots to fill, it's immediately ready for fulfillment. Based on the intent configuration, it returns the parameter to the chat client, Intent Help is ReadyForFulfillment.

  1. Finally, test the OrderProduct intent by typing Because the productFlavor and orderQuantity slots are not filled at this stage, the bot should prompt you to fill them.

Additionally, after filling all the slots, the bot should prompt you for an order confirmation, as configured in task 5. If you answer the confirmation prompt with , your intent never reaches the "ready for fulfillment" state, and Amazon Lex ends the session.

You might notice that you can order any flavor in the world, even if it is not part of the productFlavor slot. This is because the slot value's are not validated yet, as you haven't configured a slot validation and fulfillment yet. This will be addressed in part 2.
