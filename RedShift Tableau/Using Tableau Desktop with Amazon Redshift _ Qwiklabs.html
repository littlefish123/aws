Overview

This lab will give you a basic understanding of using Tableau with Amazon Redshift for visualizing data. In this lab, you will build a dashboard using Tableau Desktop by connecting it to Amazon Redshift, and using the data to create a visualization.

Note: registration and providing your personal contact information to Tableau is required for access to the trial version of Tableau Desktop needed for this lab. You may be contacted by Tableau as per their license agreements, which are provided during installation.

Topics covered

By the end of this lab you will learn how to:

  • Load data into Amazon Redshift from Amazon S3
  • Connect to an Amazon Redshift cluster from Tableau
  • Create a dashboard using Tableau

Lab Prerequisites

Familiarity with Data-Warehousing and Business-intelligence concepts and basic knowledge of Amazon Redshift is required for this lab. If you’re not familiar with Amazon Redshift, then you can take the introductory lab “Introduction to Amazon Redshift”. Amazon Elastic Compute Cloud (Amazon EC2) provides scalable computing capacity in the Amazon Web Services (AWS) cloud. Using Amazon EC2 eliminates your need to invest in hardware up front, so you can develop and deploy applications faster. You can use Amazon EC2 to launch as many or as few virtual servers as you need, configure security and networking, and manage storage. Amazon EC2 enables you to scale up or down to handle changes in requirements or spikes in popularity, reducing your need to forecast traffic.

Task 2: Load data in Amazon Redshift from Amazon S3

Amazon Redshift: Amazon Redshift is a fast, scalable data warehouse that makes it simple and cost-effective to analyze all your data across your data warehouse and data lake. Redshift delivers ten times faster performance than other data warehouses by using machine learning, massively parallel query execution, and columnar storage on high-performance disk. You can setup and deploy a new data warehouse in minutes, and run queries across petabytes of data in your Redshift data warehouse, and exabytes of data in your data lake built on Amazon S3. You can start small for just $0.25 per hour and scale to $250 per terabyte per year, less than one-tenth the cost of other solutions.

Amazon S3: Amazon Simple Storage Service (Amazon S3) is storage for the Internet. You can use Amazon S3 to store and retrieve any amount of data at any time, from anywhere on the web. You can accomplish these tasks using the simple and intuitive web interface of the AWS Management Console.

The data we will use for this lab is about the commercial flights in the United States between Years 1998-2008 from FAA. The data is split into these 3 tables

  • fact_flights – gives you information about every flights including its origin and destination airports, and scheduled and actual times
  • dim_airports – gives you information about the airports, and has two columns origin_airport_code and destination_airport_code
  • dim_carriers – gives you information about the Airlines, like carrier name

You will load these tables with the data available in Amazon S3 as objects.

On the Windows server desktop, you should see a power-shell script - run-sql. This PowerShell scripts internally runs two SQL scripts – create-table.sql and load-data.sql to create the tables and load the data from Amazon S3 into these tables. You will use the PowerShell scripts and the ODBC interface here to connect to Amazon Redshift, and run the SQL commands. You can also use a tool like – Aginity Workbench for Amazon Redshift or SQL Workbench/J to connect to Amazon Redshift.

  1. Right-click on the run-sql icon, and select Run with PowerShell.

  2. If prompted, Do you want to change the execution policy?

  • Enter
  • Press Enter

The script will start running in a new window and will print successful message once the tables are created and data has been loaded. This script may take a few minutes to load data into Redshift, since it needs to insert around 40 Million rows from Amazon S3 into the Redshift table. While the script is loading, you can start with the next steps to install and activate Tableau.

  1. Double-click the fact_flights table.

  1. Double-click the dim_carriers table.

Tableau will automatically make a relationship between the tables based on columns name and define the right joins.

Building a dashboard

  1. Create a new dashboard, by clicking on the New Dashboard button.
